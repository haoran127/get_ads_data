# 📖 受众定向技术

> 精准投放的核心技术

---

## 🎯 什么是受众定向？

**受众定向**（Audience Targeting）是通过分析用户的特征和行为，将广告精准投放给最有可能感兴趣的人群。

### **核心目标**

$$提升 eCPM = 提升 CTR \times 稳定 bid$$

受众定向的本质是**提升广告与用户的相关性**，从而提高点击率。

## 📊 定向类型分类

### **按信息来源分类**

```
┌─────────────────────────────────────────────────────────────────┐
│                    受众定向类型                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   t(u) - 用户标签                                               │
│   └── 基于用户历史行为                                           │
│   └── 如：兴趣定向、人口属性定向                                  │
│                                                                 │
│   t(c) - 上下文标签                                              │
│   └── 基于当前访问环境                                           │
│   └── 如：地域定向、内容定向                                      │
│                                                                 │
│   t(a,u) - 定制化标签                                            │
│   └── 结合广告主数据和用户行为                                    │
│   └── 如：重定向、相似人群扩展                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### **按定向维度分类**

| 类型 | 说明 | 示例 |
|-----|------|-----|
| **地域定向** | 按用户地理位置 | 北京市、朝阳区 |
| **时间定向** | 按投放时段 | 工作日晚间 |
| **人口属性** | 按用户基本特征 | 25-35岁女性 |
| **兴趣定向** | 按用户兴趣爱好 | 汽车爱好者 |
| **行为定向** | 按历史行为 | 近30天浏览过汽车 |
| **设备定向** | 按设备类型 | iPhone用户 |
| **重定向** | 访客找回 | 访问过官网的用户 |
| **相似人群** | Look-alike | 与种子用户相似 |

│                                                                 │

│                                                                 │
│   学历：     高中以下 / 大专 / 本科 / 研究生及以上                  │
│                                                                 │
│   收入：     低收入 / 中等收入 / 高收入                           │
│                                                                 │
│   婚姻：     单身 / 恋爱中 / 已婚 / 有子女                        │
│                                                                 │
│   职业：     学生 / 白领 / 蓝领 / 自由职业 / 企业主                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### **属性推断方法**

```
如何知道用户的性别？

1. 直接获取
   └── 用户注册时填写
   └── 社交账号授权

2. 行为推断
   └── 浏览内容：美妆、母婴 → 女性概率高
   └── 搜索记录：剃须刀、男装 → 男性概率高
   └── 购买记录：女装、化妆品 → 女性概率高

3. 设备信息
   └── 机型偏好（某些手机品牌有性别倾向）
   └── App安装（特定App的用户性别分布）

4. 预测模型
   └── 输入：多维度行为特征
   └── 输出：性别概率（如：女性70%，男性30%）
```

## 🎯 兴趣定向

### **兴趣标签体系**

```
兴趣标签层级结构示例：

汽车
├── 汽车资讯
│   ├── 新车上市
│   ├── 汽车评测
│   └── 行业动态
├── 购车意向
│   ├── 比较车型
│   ├── 询价
│   └── 4S店查询
├── 车型偏好
│   ├── SUV
│   ├── 轿车
│   ├── 新能源
│   └── 豪华品牌
└── 汽车后市场
    ├── 维修保养
    ├── 汽车用品
    └── 二手车
```

### **兴趣标签生成方法**

| 方法 | 说明 | 优缺点 |
|-----|------|-------|
| **规则定义** | 人工定义分类规则 | 准确但覆盖有限 |
| **内容分析** | 分析用户浏览内容 | 覆盖广但有噪音 |
| **行为聚类** | 基于行为相似性聚类 | 自动化但难解释 |
| **主题模型** | LDA等算法提取主题 | 自动化+可解释 |

## 🔄 行为定向

### **什么是行为定向？**

**行为定向**（Behavioral Targeting, BT）是根据用户历史的网络行为来推断其兴趣，进行精准投放。

### **行为类型**

```
┌─────────────────────────────────────────────────────────────────┐
│                    用户行为类型                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   📱 浏览行为                                                    │
│      └── 访问了哪些网页/App                                      │
│      └── 停留时长、滚动深度                                      │
│                                                                 │
│   🔍 搜索行为                                                    │
│      └── 搜索了什么关键词                                        │
│      └── 点击了哪些结果                                          │
│                                                                 │
│   🛒 购买行为                                                    │
│      └── 浏览/收藏/加购/购买了什么商品                           │
│      └── 购买金额、频次                                          │
│                                                                 │
│   📺 内容消费                                                    │
│      └── 观看了什么视频                                          │
│      └── 阅读了什么文章                                          │
│                                                                 │
│   📱 App使用                                                     │
│      └── 安装了哪些App                                           │
│      └── 使用频率和时长                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### **行为数据价值排序**

```
从行为到意图的价值链：

低价值                                               高价值
   │                                                   │
   ▼                                                   ▼
页面浏览 ──▶ 内容互动 ──▶ 搜索行为 ──▶ 加购/收藏 ──▶ 购买行为
   │           │           │           │           │
  意图        兴趣       明确需求    强购买意向     已转化
  模糊        萌芽
```

### **行为时效性**

```
行为随时间的价值衰减：

价值
  │
  │ ████
  │ ████████
  │ ████████████
  │ ████████████████
  │ ████████████████████
  └───────────────────────────▶ 时间
    1天  3天  7天  14天  30天

建议策略：
• 1-3天内的行为：高权重
• 7天内的行为：中权重
• 30天内的行为：低权重
• 30天以上：考虑不使用
```

## 🎯 重定向（Retargeting）

### **什么是重定向？**

**重定向**是指对已经与广告主有过互动（如访问网站、浏览商品）的用户进行再次广告投放。

### **重定向类型**

```
┌─────────────────────────────────────────────────────────────────┐
│                    重定向类型                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   🏠 站点重定向                                                  │
│      └── 访问过官网的用户                                        │
│      └── 访问过特定页面的用户                                    │
│                                                                 │
│   🛍️ 商品重定向                                                  │
│      └── 浏览过某商品的用户                                      │
│      └── 加入购物车未购买的用户                                  │
│      └── 购买过某品类的用户                                      │
│                                                                 │
│   📧 CRM重定向                                                   │
│      └── 上传客户邮箱/手机号匹配                                 │
│      └── 老客户召回                                              │
│                                                                 │
│   📱 App重定向                                                   │
│      └── App内行为用户                                           │
│      └── 卸载用户召回                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### **重定向策略矩阵**

```
用户分群与策略：

                    购买意向
                低  ──────────▶  高
           ┌────────────────────────────┐
           │   浏览用户    │  加购用户   │  低
  距今     │   品牌曝光    │  优惠刺激   │   │
  时间     ├────────────────────────────┤   │
           │   弃购用户    │  复购用户   │   ▼
           │   紧急挽回    │  交叉销售   │  高
           └────────────────────────────┘
```

### **重定向效果数据**

| 用户类型 | 转化率对比 | 建议出价 |
|---------|-----------|---------|
| 普通用户 | 1x | 基准价 |
| 浏览用户 | 2-3x | 1.2-1.5倍 |
| 加购用户 | 5-8x | 2-3倍 |
| 弃购用户 | 3-5x | 1.5-2倍 |

## 👥 Look-alike（相似人群扩展）

### **什么是 Look-alike？**

**Look-alike** 是基于已有的种子用户，找到与之相似的潜在用户进行投放。

### **Look-alike 流程**

```
┌─────────────────────────────────────────────────────────────────┐
│                  Look-alike 工作原理                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Step 1: 定义种子用户                                          │
│           └── 高价值客户、购买用户、注册用户...                   │
│                                                                 │
│   Step 2: 分析种子特征                                          │
│           └── 人口属性分布                                       │
│           └── 兴趣偏好                                          │
│           └── 行为模式                                          │
│                                                                 │
│   Step 3: 在大盘用户中匹配                                       │
│           └── 计算每个用户与种子的相似度                          │
│           └── 按相似度排序                                       │
│                                                                 │
│   Step 4: 输出扩展人群                                          │
│           └── Top 1% → 高相似度，量小                            │
│           └── Top 5% → 中高相似度                               │
│           └── Top 10% → 中等相似度，量大                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### **相似度 vs 覆盖量 权衡**

```
相似度
  │
  │ ●
  │   ●
  │      ●
  │          ●
  │               ●
  │                     ●
  └───────────────────────────────▶ 扩展比例
    1%   3%   5%   10%  15%  20%

扩展比例越大，覆盖量越大，但相似度下降
需要根据预算和目标选择合适的扩展比例
```

## 🌐 上下文定向

### **什么是上下文定向？**

**上下文定向**是根据**广告展示时的页面内容**进行广告匹配。

### **上下文定向方法**

```
┌─────────────────────────────────────────────────────────────────┐
│                    上下文定向方法                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   1. URL/频道定向                                               │
│      └── auto.sohu.com → 汽车频道 → 投放汽车广告                 │
│                                                                 │
│   2. 关键词定向                                                  │
│      └── 页面包含"新能源汽车"关键词 → 投放相关广告               │
│                                                                 │
│   3. 主题模型                                                    │
│      └── 用NLP分析页面主题 → 匹配相关广告                        │
│                                                                 │
│   4. 图片/视频分析                                               │
│      └── 识别页面中的图片内容 → 匹配相关广告                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### **上下文定向 vs 行为定向**

| 维度 | 上下文定向 | 行为定向 |
|-----|-----------|---------|
| **数据来源** | 当前页面内容 | 历史行为数据 |
| **隐私风险** | 低（不需用户数据） | 高（需要追踪） |
| **精准度** | 中等 | 较高 |
| **适用场景** | 品牌安全敏感 | 效果导向 |

## 📈 定向效果评估

### **Reach/CTR 曲线**

```
评估定向质量的核心工具：

CTR
  │
  │ ●
  │   ●
  │      ●
  │          ●
  │               ●
  │                     ●
  │                            ●
  └───────────────────────────────────────▶ Reach (覆盖人群比例)
    1%   5%   10%   20%   30%   50%   100%

理想曲线：
• 小范围人群CTR很高（精准）
• 随着扩大范围，CTR平滑下降
• 不应出现曲线上升的异常情况
```

### **定向组合策略**

```
示例：某电商618大促定向策略

┌─────────────────────────────────────────────────────────────────┐

│                                                                 │
│   层级2：高意向人群                                              │

## 🎓 本章小结

### **核心要点**

1. **定向类型**：用户标签、上下文标签、定制化标签
2. **关键定向**：地域、人口属性、兴趣、行为、重定向
3. **Look-alike**：基于种子用户扩展相似人群
4. **效果评估**：Reach/CTR曲线

### **定向策略原则**

| 原则 | 说明 |
|-----|------|
| **漏斗分层** | 高意向人群优先，逐层扩展 |
| **数据时效** | 近期行为权重更高 |
| **组合使用** | 多种定向方式组合 |
| **持续优化** | 根据效果数据调整 |

### **思考题**

1. 为什么重定向的转化率最高？
2. Look-alike 扩展比例如何选择？
3. 隐私保护趋势下，定向技术如何演进？
