# 📖 合约广告

> 品牌广告的主要交易方式

---

## 🎯 什么是合约广告？

**合约广告**是指广告主与媒体之间通过**预先签订合同**的方式进行广告交易，合同中约定广告投放的量（展示次数或时间）和价格。

### **核心特点**

| 特点 | 说明 |
|-----|------|
| **预先约定** | 投放前确定量和价格 |
| **量的保证** | 媒体承诺完成约定的展示量 |
| **品牌溢价** | 可以获得优质位置的独占权 |
| **计划性强** | 适合大型品牌活动的排期 |

---

## 📊 合约广告的类型

### **1. CPT 广告（Cost Per Time）**

**按时间段包断**：广告主购买某个广告位在特定时间段内的独占展示权。

```
┌─────────────────────────────────────────┐
│             门户网站首页                   │
├─────────────────────────────────────────┤
│                                         │
│    ┌─────────────────────────┐         │
│    │      首屏Banner位        │         │
│    │                         │         │
│    │    某汽车品牌独占        │         │
│    │    2024.1.1-1.7         │         │
│    │                         │         │
│    └─────────────────────────┘         │
│                                         │
└─────────────────────────────────────────┘

价格示例：首页Banner位 7天 = ¥500万
```

**特点**：
- ✅ 完全独占，品牌曝光强
- ✅ 适合大型品牌活动
- ❌ 价格昂贵
- ❌ 无法精准定向

### **2. CPM 广告（Cost Per Mille）**

**按展示量合约**：广告主购买一定数量的广告展示。

```
合约示例：
─────────────────────────────────────────
广告主：某手机品牌
目标受众：25-40岁男性用户
展示量：1000万次曝光
单价：¥30 CPM
总价：¥30 × 10,000 = ¥30万
投放周期：2024.3.1-3.15
─────────────────────────────────────────
```

**特点**：
- ✅ 可以进行受众定向
- ✅ 价格相对灵活
- ✅ 适合品牌曝光目标
- ❌ 需要复杂的在线分配算法保证合约

---

## 🔄 合约广告 vs 竞价广告

| 对比维度 | 合约广告 | 竞价广告 |
|---------|---------|---------|
| **交易方式** | 预先约定 | 实时竞价 |
| **价格确定** | 销售谈判 | 市场竞争 |
| **量的保证** | 有保证 | 无保证 |
| **适用场景** | 品牌广告 | 效果广告 |
| **广告主类型** | 大型品牌 | 中小广告主 |
| **结算方式** | CPT/CPM | CPC/CPA |
| **定向能力** | 相对较弱 | 非常精细 |

## ⚙️ 合约广告的核心技术问题

### **在线分配问题（Online Allocation）**

**问题描述**：如何在实时的广告请求中，合理分配各个合约的展示机会，使得所有合约都能在截止日期前完成？

```
场景示例：

合约A：女性用户，需要100万展示，截止3月15日
合约B：北京用户，需要50万展示，截止3月10日
合约C：25-35岁用户，需要80万展示，截止3月20日

当一个【北京的25岁女性用户】访问时，
这次展示应该分配给哪个合约？

需要考虑：
1. 各合约的剩余量
2. 各合约的截止时间
3. 各合约之间的受众重叠
4. 未来流量的预测
```

### **流量预测**

为了做好在线分配，需要预测未来的流量分布：

```
预测维度：
├── 时间维度：每小时/每天的流量变化
├── 地域维度：各城市的流量分布
├── 人群维度：各类人群的流量占比
└── 设备维度：PC/移动的流量分布
```

### **HWM 算法（High Water Mark）**

一种常用的在线分配算法：

```
HWM 算法核心思想：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 计算各合约的"紧迫程度"
   紧迫程度 = 剩余需求量 / 可用流量

2. 按紧迫程度排序合约

3. 为每个合约分配流量：
   - 优先满足最紧迫的合约
   - 计算每个合约的分配比例

4. 在线投放时：
   - 按优先级检查各合约
   - 按分配比例决定是否投放
```

│      │                  │                │     │

## 💰 合约广告的计费模式

### **CPT（Cost Per Time）**

```
计费公式：
总价 = 广告位日均价格 × 投放天数

示例：
某门户首页焦点图
- 日均价格：¥100万/天
- 投放7天
- 总价 = 100万 × 7 = ¥700万
```

### **CPM（Cost Per Mille）**

```
计费公式：
总价 = (展示量 / 1000) × CPM单价

示例：
- 目标展示量：1000万次
- CPM单价：¥50
- 总价 = (10,000,000 / 1000) × 50 = ¥50万
```

## 📊 合约广告的效果评估

### **品牌类指标**

| 指标 | 计算方式 | 说明 |
|-----|---------|------|
| **Reach（触达）** | 去重用户数 | 有多少人看到了广告 |
| **Frequency（频次）** | 总展示/触达人数 | 平均每人看到几次 |
| **GRP（毛评点）** | Reach% × Frequency | 品牌传播强度 |
| **Viewability** | 可视展示/总展示 | 广告是否真正被看到 |

## ⚠️ 合约广告的挑战

### **1. 合约冲突**

当多个合约的目标受众重叠时，可能出现无法同时满足的情况。

```
示例：
合约A：女性用户 200万展示
合约B：北京用户 150万展示
合约C：女性+北京用户 100万展示

如果北京女性用户的总流量只有300万，
则这三个合约可能无法同时完成。
```

### **2. 流量波动**

实际流量可能与预测存在偏差，影响合约完成。

### **3. 品牌安全**

需要避免广告出现在不当内容旁边。
