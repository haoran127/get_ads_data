# 📖 原生广告与移动广告

> 内容即广告的新形态

---

## 🎯 什么是原生广告？

**原生广告**（Native Advertising）是指在形式和功能上与其所在平台内容**融为一体**的广告形式。

### **原生广告的核心特征**

```
传统展示广告 vs 原生广告

┌─────────────────────────────────────────────────────────────────┐
│               传统展示广告                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌──────────────────────────────────┐  ┌──────────────────┐   │
│   │                                  │  │ 🔲 广告Banner    │   │
│   │         内容区域                  │  │                  │   │
│   │                                  │  │   与内容分离     │   │
│   │                                  │  │   明显的边界     │   │
│   └──────────────────────────────────┘  └──────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                 原生广告                                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌──────────────────────────────────────────────────────────┐ │
│   │  内容1                                                    │ │
│   ├──────────────────────────────────────────────────────────┤ │
│   │  内容2                                                    │ │
│   ├──────────────────────────────────────────────────────────┤ │
│   │  [广告] 推广内容 ← 融入信息流，形式与内容一致              │ │
│   ├──────────────────────────────────────────────────────────┤ │
│   │  内容3                                                    │ │
│   └──────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### **原生的两个维度**

| 维度 | 说明 | 示例 |
|-----|------|-----|
| **表现原生** | 广告的展示形式与内容一致 | 信息流广告 |
| **意图原生** | 广告的触发与用户意图匹配 | 搜索广告 |

## 📱 信息流广告

### **什么是信息流广告？**

信息流广告是在**社交媒体、资讯App等信息流**中插入的广告，是原生广告最主要的形式。

### **信息流广告示例**

```

│                                                                 │
│  [广告] 某汽车品牌                              广告            │
│  新款SUV震撼上市，限时优惠...                                   │
│  [图片/视频]                                                    │
│  了解更多 ▶                                                     │
│  ❤️ 赞  💬 评论  ↗️ 分享                                        │
```

### **主流信息流广告平台**

| 平台 | 特点 | 定向能力 | 适合场景 |
|-----|------|---------|---------|
| **微信朋友圈** | 社交属性强，用户质量高 | 精准 | 品牌、电商 |
| **抖音/快手** | 视频为主，用户时长长 | 兴趣定向 | 品牌、游戏、电商 |
| **今日头条** | 资讯为主，算法推荐 | 阅读兴趣 | 信息类广告 |
| **微博** | 热点传播，KOL效应 | 社交关系 | 品牌营销 |
| **小红书** | 种草社区，女性用户多 | 兴趣标签 | 美妆、生活方式 |

```
│                                                                 │
│   🕐 全天候触达                                                  │
│      └── 用户随时携带设备                                        │
│      └── 可基于时间和场景投放                                    │
```

### **移动广告的挑战**

| 挑战 | 说明 | 解决方案 |
|-----|------|---------|
| **屏幕小** | 广告位空间有限 | 原生广告、信息流广告 |
| **数据割裂** | App间数据不互通 | SDK、设备ID打通 |
| **用户体验** | 误点击问题严重 | 优化广告位设计 |
| **归因困难** | 跨App转化追踪难 | 第三方归因平台 |

## 🎨 移动广告形式

### **1. 横幅广告（Banner）**

```
┌─────────────────────────────────────────┐
│                                         │
│              App 内容                    │
│                                         │
│                                         │
├─────────────────────────────────────────┤
│        广告 Banner                       │
│     某品牌 - 点击了解更多                 │
└─────────────────────────────────────────┘

特点：位置固定，展示持续
问题：点击率低，误点击多
```

### **2. 插屏广告（Interstitial）**

```
┌─────────────────────────────────────────┐
│                                         │
│     ╔═══════════════════════════════╗   │
│     ║                               ║   │
│     ║         广告内容               ║   │
│     ║                               ║   │
│     ║      某游戏 - 立即下载         ║   │
│     ║                               ║   │
│     ║         [关闭 X]               ║   │
│     ╚═══════════════════════════════╝   │
│                                         │
└─────────────────────────────────────────┘

出现时机：关卡间、页面切换时
特点：强曝光，但可能影响体验
```

### **3. 开屏广告（Splash）**

```
┌─────────────────────────────────────────┐
│                                         │
│                                         │
│           品牌广告画面                   │
│                                         │
│                                         │
│                                         │
│           跳过 5s                        │
│                                         │
└─────────────────────────────────────────┘

出现时机：App启动时
特点：100%可见度，品牌价值高
```

### **4. 激励视频广告（Rewarded Video）**

```
┌─────────────────────────────────────────┐
│                                         │
│    看广告可获得 💎 50钻石                 │
│                                         │
│    ╔═════════════════════════════════╗  │
│    ║                                 ║  │
│    ║        视频广告播放中            ║  │
│    ║                                 ║  │
│    ║                                 ║  │
│    ╚═════════════════════════════════╝  │
│                                         │
│    进度：████████░░░░░ 15s/30s          │
│                                         │
└─────────────────────────────────────────┘

特点：用户主动选择观看，转化意愿高
适用：游戏、工具类App
```

### **广告形式对比**

| 形式 | CTR | CPM | 用户体验 | 适用场景 |
|-----|-----|-----|---------|---------|
| 横幅 | 低 | 低 | 影响小 | 长期品牌曝光 |
| 插屏 | 中 | 中 | 有打扰 | 游戏、工具 |
| 开屏 | 高 | 高 | 可接受 | 品牌广告 |
| 激励视频 | 极高 | 高 | 正向 | 游戏、增值 |
| 信息流 | 中高 | 中高 | 原生融合 | 效果广告 |

## 🎯 原生广告平台设计

### **原生广告的挑战**

```
传统广告平台：
广告主 ──▶ 上传创意 ──▶ 投放 ──▶ 所有媒体显示相同创意

原生广告挑战：
• 每个媒体的内容形式不同
• 广告需要"融入"各自的环境
• 无法使用统一的创意模板
```

### **解决方案：结构化广告内容**

```
┌─────────────────────────────────────────────────────────────────┐
│              原生广告的结构化数据                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   广告主提供结构化素材：                                          │
│                                                                 │
│   {                                                             │
│     "title": "品牌新品上市",                                     │
│     "description": "限时优惠，立即抢购...",                       │
│     "icon": "logo.png",                                         │
│     "main_image": "product.jpg",                                │
│     "cta": "立即购买",                                           │
│     "landing_url": "https://..."                                │
│   }                                                             │
│                                                                 │
│   媒体根据自身样式渲染：                                          │
│                                                                 │
│   媒体A（资讯流）：      媒体B（社交流）：      媒体C（电商流）：   │
│   ┌──────────────┐    ┌──────────────┐    ┌──────────────┐    │
│   │ [icon] title │    │ 🏷️ 广告       │    │  💰 推广商品  │    │
│   │ description  │    │ [main_image] │    │ [main_image] │    │
│   │ [main_image] │    │ title        │    │ title        │    │
│   │ [cta ▶]      │    │ [cta]        │    │ ¥99 [cta]    │    │
│   └──────────────┘    └──────────────┘    └──────────────┘    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 📊 原生广告的效果优势

### **为什么原生广告效果更好？**

```
Banner盲视现象：

用户已经学会了"忽略"传统广告位
研究表明，用户会无意识地跳过页面边缘的广告区域

原生广告规避了这个问题：
• 出现在用户的主要内容消费路径上
• 形式与内容一致，不会被自动忽略
• 如果内容相关，可能被主动关注
```

### **效果数据对比**

| 指标 | 传统Banner | 原生广告 | 提升 |
|-----|-----------|---------|-----|
| CTR | 0.1% | 0.5-1% | 5-10倍 |
| 品牌记忆 | 15% | 32% | 2倍+ |
| 购买意向 | 8% | 18% | 2倍+ |
| 可视度 | 30% | 75% | 2.5倍 |

## ⚠️ 原生广告的争议

### **透明性问题**

```
争议：原生广告是否在"欺骗"用户？

观点1（批评）：
• 广告伪装成内容，误导用户
• 用户可能无法区分广告和内容
• 损害媒体公信力

观点2（支持）：
• 有明确的"广告"标识
• 提供了有价值的信息
• 比强制打断式广告体验更好

行业规范：
• 必须明确标注"广告"/"推广"
• 不能模仿新闻报道格式
• 落地页要与广告内容一致
```

### **最佳实践**

| 做法 | 建议 |
|-----|------|
| **标识清晰** | 明显标注"广告"字样 |
| **内容相关** | 广告内容与页面主题相关 |
| **价值导向** | 提供有用的信息而非纯促销 |
| **体验一致** | 落地页承诺与广告一致 |

## 📈 移动广告的归因

### **归因是什么？**

**归因**（Attribution）是确定用户转化（如下载、购买）应归功于哪个广告渠道的过程。

### **移动归因的挑战**

```
场景：用户在A平台看到广告，在应用商店下载App

问题：
用户在A平台点击广告
     │
     ▼
跳转到应用商店
     │
     ▼
下载并打开App
     │
     ▼
如何知道这个用户来自A平台？

解决：设备ID匹配
• iOS: IDFA（受限）→ SKAN
• Android: GAID
• 第三方归因平台：AppsFlyer, Adjust等
```

### **常见归因模型**

| 模型 | 说明 | 适用场景 |
|-----|------|---------|
| **最后点击** | 归功于转化前最后一次点击 | 效果类广告 |
| **首次点击** | 归功于首次接触的渠道 | 品牌认知 |
| **线性归因** | 所有接触点平均分配 | 多触点旅程 |
| **时间衰减** | 越接近转化权重越高 | 长决策周期 |

## 📱 iOS 隐私变化的影响

### **ATT（App Tracking Transparency）**

```
iOS 14.5+ 的变化：

之前：App可自动获取IDFA进行跨应用追踪
之后：必须弹窗请求用户授权

┌─────────────────────────────────────────┐
│                                         │
│   "XXX App"想要追踪您在                  │
│   其他公司App和网站上的活动              │
│                                         │
│   ┌─────────────────────────────────┐   │
│   │         允许追踪                 │   │
│   └─────────────────────────────────┘   │
│                                         │
│   ┌─────────────────────────────────┐   │
│   │       要求App不追踪              │   │
│   └─────────────────────────────────┘   │
│                                         │
└─────────────────────────────────────────┘

影响：约80%用户选择"不追踪"
```

### **SKAN（SKAdNetwork）**

苹果提供的隐私保护归因方案：

| 特点 | 说明 |
|-----|------|
| **延迟归因** | 归因数据延迟24-48小时 |
| **有限数据** | 只提供聚合级别数据 |
| **转化值** | 只能传递0-63的数值 |
| **无用户级数据** | 无法追踪具体用户 |
